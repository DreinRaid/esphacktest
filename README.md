# ESP-HACK Wiki (статический сайт)

Вики в виде HTML/CSS/JS. Публикация — **GitHub Pages**.

---

## GitHub Pages: с нуля

### 1. Репозиторий на GitHub

1. Зайди на [github.com](https://github.com), **New repository**.
2. Имя репозитория, например `ESP-HACK` (для адреса сайта важно имя — см. шаг 6).
3. Создай репозиторий (можно без README, если зальёшь файлы вики с компьютера).

### 2. Загрузить файлы сайта

В корень репозитория должны попасть **`index.html`**, папки **`css/`**, **`js/`**, **`data/`**, **`en/`**, **`images/`** и т.д. — как в этом проекте.

Варианты:

- **Через сайт GitHub:** Add file → Upload files.
- **Через Git на компьютере:**
  ```bash
  git clone https://github.com/ТВОЙ_ЛОГИН/ИМЯ_РЕПО.git
  cd ИМЯ_РЕПО
  # скопируй сюда файлы вики
  git add .
  git commit -m "Add wiki site"
  git push origin main
  ```
  Если у репозитория ветка **`master`**, замени `main` на `master` (workflow ниже срабатывает на обе ветки).

### 3. Файлы для Pages в этом проекте

| Файл | Зачем |
|------|--------|
| **`.nojekyll`** | Отключает Jekyll: GitHub отдаёт файлы как есть, без лишней обработки. |
| **`.github/workflows/pages.yml`** | После каждого push в `main` или `master` собирает сайт и публикует его на Pages. |

Если сайт лежит **не в корне**, а в подпапке (например `docs/`), в `pages.yml` в шаге **Upload artifact** поменяй:

```yaml
path: "docs"
```

(вместо `path: "."`).

### 4. Включить GitHub Pages в настройках репозитория

1. Репозиторий на GitHub → **Settings** (Настройки).
2. Слева **Pages**.
3. В блоке **Build and deployment**:
   - **Source:** выбери **GitHub Actions** (не «Deploy from a branch»).
4. Сохранять отдельно ничего не нужно — источник переключается сразу.

### 5. Первый запуск деплоя

1. Вкладка **Actions** → должен появиться workflow **Deploy Pages**.
2. Если после push он не запустился: **Actions** → **Deploy Pages** → **Run workflow**.
3. Дождись зелёной галочки у последнего запуска.

Если GitHub впервые спросит разрешение для **GitHub Pages** — согласись (одобрить workflow для `github-pages`).

### 6. Адрес сайта

Для обычного репозитория (не `username.github.io`):

```text
https://ТВОЙ_ЛОГИН.github.io/ИМЯ_РЕПО/
```

Пример: логин `Teapot174`, репозиторий `ESP-HACK` →  
`https://Teapot174.github.io/ESP-HACK/`

Главная страница — **`index.html`** (открывается по ссылке выше, часто без явного `/index.html`).

### 7. Как обновлять сайт

Любой **push** в ветку **`main`** или **`master`** снова запускает **Deploy Pages** и обновляет опубликованный сайт.

### 8. Свой домен (по желанию)

**Settings → Pages → Custom domain** — укажи домен и следуй подсказкам GitHub (DNS у регистратора). Файл **`CNAME`** в корне репозитория GitHub при необходимости создаст сам.

## Локальный просмотр

Открой `index.html` в браузере или подними простой сервер из папки с сайтом (чтобы корректно работали пути), например:

```bash
npx serve .
```

## Админка (Decap CMS)

Папка **`admin/`**. После публикации на Pages адрес вида  
`https://ТВОЙ_ЛОГИН.github.io/ИМЯ_РЕПО/admin/`  
Для входа через GitHub нужна настройка OAuth-приложения в GitHub (см. документацию Decap CMS).
